rcExecMyRule

execMyRuleInp_t *  execMyRuleInp,

Description:
Submit a user defined rule to be executed by an iRODS server.
Primary User:
client
Category:
rule operations
Since:
1.0
Author:
Mike Wan
Date:
2007
Remarks:
none

none
Example Usage:

Execute a rule that create a collection /myZone/home/john/coll1
int status;
execMyRuleInp_t execMyRuleInp;
msParamArray_t msParamArray;
msParamArray_t *outParamArray = NULL;
bzero (&execMyRuleInp, sizeof (execMyRuleInp));
rstrcpy (execMyRuleInp.myRule, "myTestRule { msiCollCreate(*Path,"0",*Status); writeLine("stdout","Create collection *Path"); }", META_STR_LEN);
memset (&msParamArray, 0, sizeof (msParamArray));
execMyRuleInp.inpParamArray =
addMsParamToArray (&msParamArray, "Path", STR_MS_T, "/myZone/home/john/coll1", 1);
rstrcpy (execMyRuleInp.outParamDesc, "ruleExecOut", LONG_NAME_LEN);
status = rcExecMyRule (conn, &execMyRuleInp, &outParamArray);
if (status < 0) {
.... handle the error
} else {
... hnadle outParamArray output
}

Parameters:
[in]conn- A rcComm_t connection handle to the server.
[in]execMyRuleInp- Elements of execMyRuleInp_t used :
  • char myRule[META_STR_LEN] - The string representing the rule to be executed..
  • rodsHostAddr_t addr - The address to execute the rule. Can be left blank.
  • char outParamDesc[LONG_NAME_LEN] - the list of output parameters separated by "%".
  • msParamArray_t *inpParamArray - Input parameters for the rule in the form of array of msParam.
  • keyValPair_t condInput - keyword/value pair input. Valid keywords:
    "looptest" - Just a test.
[out]outParamArray- A msParamArray_t containing an array of msParam_t.
