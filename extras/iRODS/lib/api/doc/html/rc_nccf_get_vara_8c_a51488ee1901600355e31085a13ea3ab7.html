rcNccfGetVara
int rcNccfGetVara ( rcComm_t *  conn,
nccfGetVarInp_t *  nccfGetVarInp,
nccfGetVarOut_t **  nccfGetVarOut 
)
Description:
libcf subsetting function. libcf function nccf_get_vara is called on the server. The iRODS software must be built with LIB_CF enabled (the line LIB_CF uncommented in config.mk).
Primary User:
client
Category:
NETCDF operations
Since:
3.1
Author:
Mike Wan
Date:
2012
Remarks:
none
Note:
none
Example Usage:

nc_open a data object and get the dimension Id of 'time':
ncOpenInp_t ncOpenInp;
int status, i;
ncInqInp_t ncInqInp;
ncInqOut_t *ncInqOut = NULL;
nccfGetVarInp_t nccfGetVarInp;
nccfGetVarOut_t *nccfGetVarOut = NULL;
float *mydata;
bzero (&ncOpenInp, sizeof (ncOpenInp));
rstrcpy (ncOpenInp.objPath, "/myZone/home/john/myfile.nc", MAX_NAME_LEN);
ncOpenInp.mode = NC_NOWRITE;
status = rcNcOpen (conn, &ncOpenInp, &ncid);
if (status < 0) {
.... handle the error
}
// inq the pressure var
bzero (&ncInqInp, sizeof (ncInqInp));
ncInqInp.ncid = ncid;
ncInqInp.paramType = NC_VAR_TYPE;
rstrcpy (ncInqInp.name, "pressure", MAX_NAME_LEN);
ncInqInp.flags = 0;
status = rcNcInq (conn, &ncInqInp, &ncInqOut);
if (status < 0) {
.... handle the error
}
// pressure subset
bzero (&nccfGetVarInp, sizeof (nccfGetVarInp));
nccfGetVarInp.ncid = ncid;
nccfGetVarInp.varid = ncInqOut->var->id;
nccfGetVarInp.latRange[0] = 30.0;
nccfGetVarInp.latRange[1] = 41.0;
nccfGetVarInp.lonRange[0] = -120.0;
nccfGetVarInp.lonRange[1] = -96.0;
nccfGetVarInp.maxOutArrayLen = 1000;
status = rcNccfGetVara (conn, &nccfGetVarInp, &nccfGetVarOut);
if (status < 0) {
.... handle the error
}
mydata = (float *) nccfGetVarOut->dataArray->buf;
printf ("pressure values: ");
for (i = 0; i < nccfGetVarOut->dataArray->len; i++) {
printf (" %f\n", mydata[i]);
}
freeNccfGetVarOut (&nccfGetVarOut);

Parameters:
[in]conn- A rcComm_t connection handle to the server.
[in]nccfGetVarInp- Elements of nccfGetVarInp_t used :
  • int ncid - the ncid from ncNcOpen.
  • int varid - the variable Id from rcNcInq.
  • int lvlIndex - A zero-based index number for the verticle level of interest (Ignored if data has no vertical axis).
  • int timestep - A zero-based index number for the timestep of interest (Ignored if data has no time axis).
  • float latRange[2] - holds the latitude start and stop values for the range of interest.
  • float holds the longitude start and stop values for the range of interest (Wrapping around the dateline is not allowed!).
[out]nccfGetVarOut- a nccfGetVarOut_t. Elements of nccfGetVarOut_t:
  • int nlat - The number of latitude values which fall within the range.
  • int nlon - The number of longitude values which fall within the range.
  • char dataType_PI[NAME_LEN] - Packing instruction of the dataType.
  • dataArray_t *dataArray - returned values of the variable. dataArray->type gives the var type; dataArray->len gives the var length; dataArray->buf contains the var values.
