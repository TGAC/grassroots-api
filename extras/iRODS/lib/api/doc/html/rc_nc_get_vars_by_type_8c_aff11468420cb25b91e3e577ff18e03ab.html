rcNcGetVarsByType
int rcNcGetVarsByType ( rcComm_t *  conn,
ncGetVarInp_t *  ncGetVarInp,
ncGetVarOut_t **  ncGetVarOut 
)
Description:
General NETCDF variable subsetting function (equivalent to nc_get_vars_type API).
Primary User:
client
Category:
NETCDF operations
Since:
3.1
Author:
Mike Wan
Date:
2012
Remarks:
none
Note:
none
Example Usage:

Example for this API can be rather involved. See the getSingleNcVarData function in rcNcInq.c on the usage of this API.

Parameters:
[in]conn- A rcComm_t connection handle to the server.
[in]ncGetVarInp- Elements of ncGetVarInp_t used :
  • int dataType - valid values are defined in ncGetVarsByType.h - NC_BYTE, NC_CHAR, NC_SHORT, NC_INT, NC_FLOAT, NC_DOUBLE, NC_UBYTE, NC_USHORT, NC_UINT, NC_INT64, NC_UINT64 or NC_STRING.
  • int ncid - the ncid from ncNcOpen.
  • int varid - the variable Id from rcNcInq or rcNcInqId.
  • int ndim - number of dimensions (rank).
  • rodsLong_t *start - A vector of rodsLong_t with ndim length specifying the index in the variable where the first of the data values will be read.
  • rodsLong_t *count - A vector of rodsLong_t with ndim length specifying the number of indices selected along each dimension.
  • rodsLong_t *stride - A vector of rodsLong_t with ndim length specifying for each dimension, the interval between selected indices.
[out]ncGetVarOut- a ncGetVarOut_t. Elements of ncGetVarOut_t:
  • char dataType_PI[NAME_LEN] - Packing instruction of the dataType.
  • dataArray_t *dataArray - returned values of the variable. dataArray->type gives the var type; dataArray->len gives the var length; dataArray->buf contains the var values.
Returns:
integer
